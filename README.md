## 它是什么

一个**职场权力博弈分析器**。

输入你的职场处境，从《大明王朝1566》中四个最经典的权力角色视角，输出一份完整的策略推演：

| 视角 | 核心逻辑 | 帮你回答 |
|------|---------|---------|
| 🔮 嘉靖 | 顶层设计者 | "领导到底想要什么？" |
| 🦊 严嵩 | 生存主义者 | "怎么先保住自己？" |
| ⚔️ 胡宗宪 | 夹缝操盘手 | "多方博弈下怎么走？" |
| ⚡ 海瑞 | 破局者 | "如果选择掀桌子呢？" |

不是角色扮演，不是"朕认为"。是提取每个人物的**决策模型**，应用到你的真实场景中。

## Demo

**输入：**

> 老板在全员会上表扬了我的方案，但会后单独找我说"这个事情你先不要推了，等等看"。怎么理解？

**输出（节选）：**

#### 🔮 嘉靖视角：看透顶层意图

公开表扬 + 私下叫停，这是经典的"既要又要"信号。表扬是做给其他人看的——可能是在敲打另一个团队，也可能是在给自己留后路。叫停是真实意图：他需要这个方案存在，但不需要它现在落地。

关键问题：是谁的利益会因为你的方案落地而受损？你老板可能不是在保护你，而是在保护他自己和那个人之间的平衡。

#### 🦊 严嵩视角：先活下来

最安全的做法：**按兵不动，但保持方案的随时可启动状态。** 不要追问"为什么不推了"——这个问题本身就是表态。把方案完善好放着，等他再提起。如果三个月他不提，说明这事已经变了，方案的功劳你已经拿到了，见好就收。

#### ⚔️ 胡宗宪视角：在夹缝中找路

你需要判断：叫停是暂时的战术需要，还是方向性否定？方法是观察后续资源配置——如果相关预算没砍、团队没调，说明只是时机问题。这时候你可以做一件事：把方案拆成几个独立的小模块，先推风险最低的那一个。如果老板没拦，说明他要的是"别一次到位"而不是"别做"。

#### ⚡ 海瑞视角：如果选择掀桌子

如果你判断这个方案对公司确实有价值，而叫停纯粹是政治原因——你有一个高风险高回报的选项：找一个非正式场合，直接问老板"是时机不对，还是方向不对？"这句话的杀伤力在于，它迫使对方给出一个明确判断，而不是继续用模糊来管理你。但前提是你已经准备好了听到任何答案。

## 安装

```bash
git clone https://github.com/xiaomengwwwww/daming-dynasty-skill ~/.claude/skills/daming-dynasty
```

或将 `SKILL.md` 手动放入你的 Claude Code Skills 目录。

## 使用

直接描述你的职场处境即可，例如：

- "两个VP在争一个新业务，都来拉我，我该站谁？"
- "我的直属领导被边缘化了，我要不要跟着他沉？"
- "新来的总监明显在架空我，但我的业绩比他好"
- "老板让我去做一个明知会失败的项目"

不需要任何特殊指令。描述你的处境，剩下的交给四位军师。

## 为什么是《大明王朝1566》

不是因为它好看（虽然它确实好看）。

是因为这部剧用 46 集讲了一件事：**在一个层层嵌套的权力结构中，不同位置的人如何在信息不对称的条件下做出最优决策。** 这不是宫斗，这是博弈论的影像化。

嘉靖教你看透老板的真实意图。严嵩教你在逆境中存活。胡宗宪教你在多方压力下执行。海瑞教你什么时候该掀桌子。

四个视角不是四种性格，是四种**位置决定的理性**。位置不同，最优策略就不同。

---

## 局限性

- 这是思维框架，不是算命。分析质量取决于你提供的信息质量。
- 适合需要策略思考的复杂场景，不适合"怎么写一封请假邮件"这种问题。
- 如果你的职场环境相对简单透明（比如外企扁平化管理），可能用不上这么重的分析。
- 四个视角有时候会给出矛盾的建议——这是特性，不是 bug。现实就是矛盾的。
